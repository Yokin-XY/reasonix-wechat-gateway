# 通过微信发送文件

当你在工作目录中生成或修改了文件，并且需要把文件发送给用户（通过微信），在回复中使用 `MEDIA:` 标签。

## 格式

在回复文本中，单独一行写文件的绝对路径，以 `MEDIA:` 开头：

```
MEDIA:/绝对路径/文件名.扩展名
```

## 示例

用户说"帮我写个快速排序，发给我":
```
我已经写好了快速排序的实现，保存在 `sort.py`。

MEDIA:/root/reasonix-workspace/sort.py
```

用户说"把当前代码打包发给我":
```
项目文件已打包。

MEDIA:/root/reasonix-workspace/project.tar.gz
```

用户说"生成一张图片":
```
已生成图片。

MEDIA:/root/reasonix-workspace/chart.png
```

## 规则

1. 路径必须是**绝对路径**（以 / 开头）
2. 每个文件占一行，一行一个 `MEDIA:` 标签
3. 多个文件就多个 `MEDIA:` 行
4. 文件路径后面不要加多余的文字
5. 文字说明写在 `MEDIA:` 行的前面或后面，不要写在同一行
6. 支持的文件类型：任意（图片、文档、代码、压缩包等）

## 触发场景

以下场景应该使用 MEDIA: 标签发送文件：

- 用户说"发给我"、"发送"、"下载"、"导出"
- 用户说"保存为文件"、"输出到文件"
- 用户说"打包"、"压缩"
- 用户要求生成代码、文档、图片等产出物
- 用户说"发到微信"、"发到手机"
- 完成了文件编辑/创建后用户要求发送结果
- 用户明确要求把工作成果发送过来

## 不需要发送的场景

- 用户只是让你写代码（没有说要发送）
- 用户只是讨论/分析代码
- 用户问问题你回答文字就行
- 文件已经在之前的对话中发过了
